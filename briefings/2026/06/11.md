## Fun fact

Ever wondered why Microsoft picked Tuesday for Patch Tuesday? When the company consolidated its scattered security fixes into one monthly release in October 2003, it chose the second Tuesday deliberately: Monday was avoided so sysadmins wouldn't return from the weekend straight into patching chaos, and a Tuesday release leaves the rest of the workweek to test and deploy fixes before everyone goes home. The rhythm became so entrenched that attackers invented "Exploit Wednesday" — reverse-engineering each patch the day after release to attack systems that hadn't updated yet. Today's record-setting 200-CVE release follows the same calendar logic that's held for over two decades. [Source](https://en.wikipedia.org/wiki/Patch_Tuesday)
